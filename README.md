This project breaks the English-only instruction bottleneck, engineering the early-fusion pipeline to process complex **Hindi instructions** to drive spatial video tracking, while utilizing a quantized Edge-LLM to generate real-time localized reasoning for segmentation shifts.

### Real-Time Explanatory Grounding
**Text Prompt (Hindi):** `"बिल्ली पेड़ पर चढ़ रही है"` *(The cat is climbing the tree)*

During the forward pass, the Edge-LLM monitors the segmentation masks. When a Common Movement Estimation (CME) threshold is exceeded (e.g., the cat shifts from running to climbing), the LLM generates autonomous reasoning for the attention shift:

> **System Explanation (Generated by Qwen2.5-1.5B):** > *"The video segmentation model initially focused on tracking an identified object based on its initial behavior patterns. However, upon detecting that the CME (Common Movement Estimation) threshold has been exceeded, it recognized a significant change in the object's movement pattern. This triggered a reevaluation of the model's attention, shifting its primary focus to identify and track a different object that matches the new observed behavior..."*

## 🔬 Architectural Novelty

Standard early-fusion RVOS models rely on English-only instruction bottlenecks and lack transparency when losing or shifting track of an object. This architecture introduces:

1. **Multilingual Spatial Tracking:** Modified the early-fusion pipeline to process complex Hindi text prompts. This localized text is routed through **RoBERTa Base** to drive the spatial video tracking of the **Segment Anything 2 (SAM2)** visual backbone.
2. **Explanatory Grounding via Edge-LLM:** Integrated **Qwen2.5-1.5B-Instruct** (4-bit Quantized via HuggingFace) directly into the inference loop to provide localized explanatory reasoning for segmentation decisions without relying on external cloud APIs.

## 🛠️ Engineering & Environment Sterilization (Cloud GPU Optimized)

Deploying legacy multi-modal architectures in modern cloud environments presents severe dependency clashes. This repository includes custom sterilization scripts to guarantee stable execution on a single **Nvidia T4 GPU (16GB VRAM)**:

* **Python 3.12 / Fairseq Survival:** The legacy `fairseq` library suffers from fatal `dataclass` mutable-default errors in modern Python. This repository utilizes custom Structure-Aware Block Patchers and a Hydra sandbox to bypass broken eager-loading loops and dynamically rebuild the configuration registry in memory.
* **Hardware-Agnostic Routing:** Stripped legacy Apple Silicon (`mps`) hardcoding from the base paper's implementation. Built dynamic CUDA interception for HuggingFace Transformers to ensure seamless execution on Linux/Nvidia clusters.
* **Memory-Safe Execution:** Implemented automated `ffmpeg` preprocessing to downscale input video tensors natively, preventing cross-attention matrices from exceeding VRAM limits during the forward pass.

## 🚀 Quick Start (One-Click Inference)

You do not need to manage dependencies manually. The entire execution environment is containerized in a Colab notebook.

1. Click the **Open In Colab** badge at the top of this repository.
2. Run the cells in order. The notebook will:
   * Clone this repository.
   * Execute the Fairseq Python 3.12 sterilization patches.
   * Download the SAM2, RoBERTa, and Qwen pre-trained weights.
   * Execute the memory-safe inference script.
3. Retrieve your final segmented video from the `demo_output/` folder.
